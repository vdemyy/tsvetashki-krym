# Объяснение: SQLite vs PostgreSQL

## 🤔 Почему была SQLite?

**SQLite** — это простая файловая база данных. Все данные хранятся в одном файле (`data/tsvetashki.db`).

### Преимущества SQLite:
- ✅ Очень простая в настройке (не нужен отдельный сервер)
- ✅ Не требует дополнительных ресурсов
- ✅ Идеальна для разработки и небольших проектов
- ✅ Данные в одном файле — легко копировать

### Недостатки SQLite:
- ❌ Плохо работает при одновременном доступе многих пользователей
- ❌ Ограниченная производительность при большой нагрузке
- ❌ Нет продвинутых функций (репликация, кластеризация)
- ❌ Проблемы с блокировками при записи

## 🚀 Почему перешли на PostgreSQL?

**PostgreSQL** — это профессиональная реляционная СУБД (система управления базами данных).

### Преимущества PostgreSQL:
- ✅ **Высокая производительность** при большой нагрузке
- ✅ **Многопользовательский доступ** — сотни пользователей одновременно
- ✅ **Надёжность** — ACID-транзакции, репликация, резервное копирование
- ✅ **Расширенные возможности**:
  - JSON/JSONB для хранения структурированных данных
  - PostGIS для геоданных (координаты мест)
  - Полнотекстовый поиск
  - Триггеры, хранимые процедуры
- ✅ **Масштабируемость** — легко увеличить мощность
- ✅ **Профессиональные инструменты** — pgAdmin, мониторинг, аналитика

### Когда нужен PostgreSQL:
- 📈 Проект растёт и нужна стабильность
- 👥 Много одновременных пользователей
- 🌍 Production-окружение (реальные пользователи)
- 🔒 Важна надёжность и безопасность данных

## 📊 Что изменилось в проекте?

### Было (SQLite):
```python
DATABASE_URL = "sqlite:///./data/tsvetashki.db"
```

Данные хранились в файле `data/tsvetashki.db`

### Стало (PostgreSQL):
```python
DATABASE_URL = "postgresql://tsvetashki:tsvetashki_pass_2026@db:5432/tsvetashki"
```

Данные хранятся в отдельном контейнере PostgreSQL

## 🔄 Как прошла миграция?

1. **Запущен PostgreSQL** в Docker контейнере
2. **Созданы таблицы** в PostgreSQL (такие же, как в SQLite)
3. **Скопированы все данные** из SQLite в PostgreSQL:
   - 29 явлений (phenomena)
   - 28 мест (places)
   - 31 событие (events)
   - Подписки и настройки Telegram бота
4. **Изменена конфигурация** приложения (`.env` файл)
5. **Проверена работоспособность** — всё работает!

## 📁 Что случилось с SQLite?

Файл `data/tsvetashki.db` **остался как резервная копия**, но приложение его больше **не использует**.

Вы можете:
- Оставить его как бэкап
- Удалить, если уверены, что PostgreSQL работает корректно

## 🛠️ Как проверить, какая БД используется?

### Способ 1: Проверить переменную окружения
```bash
docker compose exec backend python -c "import os; print('DATABASE_URL:', os.getenv('DATABASE_URL'))"
```

**Должно вывести:**
```
DATABASE_URL: postgresql://tsvetashki:tsvetashki_pass_2026@db:5432/tsvetashki
```

### Способ 2: Проверить данные
```bash
# Посмотреть данные в PostgreSQL
docker compose exec db psql -U tsvetashki -d tsvetashki -c "SELECT COUNT(*) FROM phenomena;"
```

**Должно вывести:** `29`

### Способ 3: Проверить через API
```bash
curl http://localhost:8000/api/events
```

Если данные возвращаются — значит PostgreSQL работает!

## 🔍 Как работает PostgreSQL в Docker?

### Контейнеры:
```
┌─────────────────────┐
│  tsvetashki_db      │  ← PostgreSQL сервер
│  (postgres:15)      │     Порт: 5432
└─────────────────────┘
          ↕
┌─────────────────────┐
│  tsvetashki_backend │  ← FastAPI приложение
│  (Python)           │     Порт: 8000
└─────────────────────┘
          ↕
┌─────────────────────┐
│  tsvetashki_pgadmin │  ← Веб-интерфейс для БД
│  (pgAdmin)          │     Порт: 5050
└─────────────────────┘
```

### Данные:
Данные PostgreSQL хранятся в **Docker volume** `postgres_data`.

Это значит:
- ✅ Данные сохраняются даже при перезапуске контейнера
- ✅ Данные не теряются при обновлении образа
- ❌ Данные удалятся только при `docker compose down -v`

## 🆚 Сравнение

| Характеристика | SQLite | PostgreSQL |
|----------------|--------|------------|
| **Тип** | Файловая БД | Клиент-серверная СУБД |
| **Настройка** | Очень простая | Требует сервер |
| **Производительность** | Низкая при нагрузке | Высокая |
| **Многопользовательский доступ** | Ограниченный | Отличный |
| **Размер БД** | До ~1 ГБ комфортно | Терабайты |
| **Резервное копирование** | Копирование файла | pg_dump, репликация |
| **Расширенные функции** | Базовые | JSON, геоданные, полнотекстовый поиск |
| **Использование** | Разработка, прототипы | Production, большие проекты |

## 💡 Рекомендации

### Используйте SQLite если:
- 🧪 Разрабатываете прототип
- 📱 Мобильное приложение
- 📊 Локальное хранилище данных
- 👤 Один пользователь

### Используйте PostgreSQL если:
- 🌍 Веб-приложение с пользователями
- 📈 Проект будет расти
- 👥 Много одновременных пользователей
- 🔒 Важна надёжность данных
- 🚀 Production-окружение
