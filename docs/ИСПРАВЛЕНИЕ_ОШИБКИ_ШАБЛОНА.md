# Исправление ошибки шаблона feed.html

## Дата: 21 мая 2026

### ❌ Проблема
После внесения изменений в дизайн страница ленты (главная страница `/`) выдавала **Internal Server Error 500**.

**Ошибка в логах:**
```
jinja2.exceptions.TemplateSyntaxError: Unexpected end of template. 
Jinja was looking for the following tags: 'endblock'. 
The innermost block that needs to be closed is 'block'.
```

### 🔍 Причина
В файле `templates/feed.html` блок `{% block scripts %}` был открыт внутри блока `{% block content %}`, но не был правильно закрыт. Структура была такой:

```jinja2
{% block content %}
  <!-- контент страницы -->
  {% block scripts %}
    <script>...</script>
  {% endblock %}
<!-- ОТСУТСТВОВАЛ {% endblock %} для закрытия content! -->
```

Jinja2 ожидал закрывающий тег `{% endblock %}` для блока `content`, но его не было.

### ✅ Решение
Добавлен закрывающий тег `{% endblock %}` после блока `content` и перед блоком `scripts`:

```jinja2
{% block content %}
  <!-- контент страницы -->
  {% endfor %}
</div>
{% endblock %}

{% block scripts %}
  <script>...</script>
{% endblock %}
```

Теперь структура правильная:
1. Блок `content` открывается и закрывается
2. Блок `scripts` открывается и закрывается отдельно

### 📝 Измененные файлы
- `templates/feed.html` - добавлен `{% endblock %}` после закрытия `</div>` карточек

### ✅ Результат
- Страница ленты теперь работает корректно
- HTTP статус: **200 OK**
- Все функции работают:
  - Фильтры (с кнопкой на мобильных)
  - Карточки событий
  - Кнопки "7 дней", "Шире", "Карта"
  - JavaScript для копирования и фильтров

### 🧪 Проверка
```bash
# Проверить статус страницы
curl -I http://localhost:8000/

# Должен вернуть:
HTTP/1.1 200 OK
```

### 📚 Урок
При работе с Jinja2 шаблонами важно следить за правильной вложенностью блоков:
- Каждый `{% block name %}` должен иметь соответствующий `{% endblock %}`
- Блоки не должны пересекаться
- Внутренние блоки должны быть полностью внутри внешних блоков
