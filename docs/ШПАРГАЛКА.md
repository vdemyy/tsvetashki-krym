## 📊 Работа с БД

### Через pgAdmin (веб-интерфейс)
1. Откройте http://localhost:5050
2. Войдите (`admin@admin.com` / `admin`)
3. Добавьте сервер (см. `ПОДКЛЮЧЕНИЕ_PGADMIN.md`)

### Через командную строку
```bash
# Подключиться к PostgreSQL
docker compose exec db psql -U tsvetashki -d tsvetashki

# Посмотреть таблицы
\dt

# Посмотреть данные
SELECT * FROM phenomena;

# Выйти
\q
```

### Быстрые запросы
```bash
# Количество записей
docker compose exec db psql -U tsvetashki -d tsvetashki -c "SELECT 'phenomena' as table, COUNT(*) FROM phenomena UNION ALL SELECT 'places', COUNT(*) FROM places UNION ALL SELECT 'events', COUNT(*) FROM events;"

# Последние события
docker compose exec db psql -U tsvetashki -d tsvetashki -c "SELECT e.id, p.name as phenomenon, pl.name as place, e.start_date FROM events e JOIN phenomena p ON e.phenomenon_id = p.id JOIN places pl ON e.place_id = pl.id ORDER BY e.start_date DESC LIMIT 5;"
```

## 💾 Резервное копирование

### Создать бэкап
```bash
# Через Docker
docker compose exec db pg_dump -U tsvetashki tsvetashki > backup_$(date +%Y%m%d_%H%M%S).sql

# Или через pgAdmin
# Правой кнопкой на базу → Backup...
```

### Восстановить бэкап
```bash
# Через Docker
docker compose exec -T db psql -U tsvetashki tsvetashki < backup_20260521_120000.sql

# Или через pgAdmin
# Правой кнопкой на базу → Restore...
```

## 🐛 Устранение проблем

### Сайт не открывается
```bash
# Проверить статус
docker compose ps

# Посмотреть логи
docker compose logs backend

# Перезапустить
docker compose restart backend
```

### БД не работает
```bash
# Проверить PostgreSQL
docker compose ps db

# Посмотреть логи
docker compose logs db

# Проверить подключение
docker compose exec db pg_isready -U tsvetashki
```

### pgAdmin не открывается
```bash
# Проверить статус
docker compose ps pgadmin

# Перезапустить
docker compose restart pgadmin

# Посмотреть логи
docker compose logs pgadmin
```

### Порт занят
```bash
# Найти процесс на порту 8000
netstat -ano | findstr :8000

# Убить процесс (замените PID)
taskkill /PID <PID> /F

# Или измените порт в docker-compose.yml
# ports:
#   - "8001:8000"  # вместо 8000:8000
```

```
├── region
├── subregion

├── id
├── phenomenon_id → phenomena.id
├── place_id → places.id
├── start_date
├── peak_date
├── end_date
├── intensity (1-5)
└── notes
```
